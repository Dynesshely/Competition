# P1020 [NOIP 1999 提高组] 导弹拦截 — 题解

## 题目大意

给定一个导弹高度序列，计算：
1. 一套导弹拦截系统（每次拦截高度不能高于前一次）最多能拦截多少导弹
2. 拦截所有导弹最少需要配备多少套系统

## 解题思路

### 第一问：最长不升子序列（LNIS）

一套系统能拦截的导弹构成一个**非递增**子序列（每发高度 ≤ 前发高度）。第一问即求最长不升子序列的长度。

### 第二问：最少系统数 = 最长上升子序列（LIS）

**Dilworth 定理**：在偏序集中，最小链覆盖数 = 最大反链的大小。

定义偏序关系：导弹 $i \preceq j$ 当且仅当 $i < j$（$i$ 先于 $j$ 到达）且 $h_i \ge h_j$（$j$ 可被同一套系统在 $i$ 之后拦截）。

- 一条**链** = 能被同一套系统拦截的导弹序列 = 一个不升子序列
- 一条**反链** = 任意两枚导弹都不能被同一套系统拦截的集合 = 一个严格上升子序列

由 Dilworth 定理，最少系统数 = 最大反链大小 = **最长严格上升子序列的长度**。

### $O(n \log n)$ 贪心 + 二分（耐心排序）

$n \le 10^5$，必须使用 $O(n \log n)$ 算法。

**第一问 LNIS 的实现方式**：对 $-h_i$ 求最长**不降**子序列。

因为 $h_i \ge h_j \iff -h_i \le -h_j$，LNIS 就转化为了 LNDS（最长不降子序列）。

LNDS 的耐心排序：维护 `tails` 数组，`tails[k]` 表示长度为 $k+1$ 的不降子序列的最小可能末尾值。对于每个 $x$，用 `upper_bound`（找第一个 `> x` 的位置）决定放置位置。

**第二问 LIS 的实现方式**：标准 LIS 耐心排序，`tails[k]` 表示长度为 $k+1$ 的严格上升子序列的最小可能末尾值。对于每个 $x$，用 `lower_bound`（找第一个 `≥ x` 的位置）决定放置位置。

### 时间复杂度

- $O(n \log n)$，$n \le 10^5$

### 空间复杂度

- $O(n)$
