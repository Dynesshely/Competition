# P2244 [POI 2004 R2] 选举预测 题解

## 题目链接

- [洛谷 P2244](https://www.luogu.com.cn/problem/P2244)

## 题目大意

有 n 名候选人进行辩论赛，规则是每次从剩余的候选人中任选两人辩论，输者淘汰，胜者留下，直到只剩一人即为最终胜者。给定候选人间的历史胜负关系（若两人曾比赛过，则再次比赛结果不变），求所有可能成为最终胜者的候选人编号。

## 解题思路

这是一道图论问题。将每个候选人看作一个节点，有向边 u -> v 表示 u 曾经赢过 v。

核心观察：如果候选人 A 从未输给过 B，那么在某个比赛顺序下 A 有可能赢过 B（或 B 可以淘汰掉 A 的所有克星），从而 B 也有可能成为最终胜者。

算法的关键思路来源于一个经典结论：**在任意竞赛图中，出度最大的节点必定可以成为最终胜者**。更一般地，从最大出度的节点出发，用 BFS 遍历那些"未被当前节点直接击败"的节点，得到的集合就是所有可能的胜者。

具体步骤：

1. **建图**：读入每个候选人的获胜记录，构建邻接表，同时计算每个节点的出度（`chudu[i]`）。
2. **找出最大出度**：找到 max_chudu 对应的所有节点，将它们作为 BFS 的初始队列。这些节点必定可以获胜。
3. **维护链表**：用双向链表 `pr[]` 和 `af[]` 维护所有"尚未确定能否获胜"的候选人，初始包含所有 1..n。
4. **BFS 扩散**：
   - 取出队首节点 t。
   - 遍历链表中的剩余候选人 x，利用有序的邻接表跳过那些 t 直接击败过的节点。
   - 如果 x 不在 t 的邻接表中（即 t 没有赢过 x），则 x 有可能获胜，将其标记并加入队列，同时从链表中删除。
5. 输出所有被标记为可能获胜的候选人。

## 代码实现

- **快读 `read()`**：重写整数输入函数，比 `scanf` 更快，适合 n 达 1e6 的大数据量。
- **邻接表**：使用链式前向星（`h[]`, `nx[]`, `to[]`），按出度编号的逆序（由于读入顺序）存储。
- **双向链表 `pr[]` / `af[]`**：初始设置 `pr[i] = i - 1`, `af[i] = i + 1`，`n+1` 作为哨兵节点。
- **`del(nd)`**：从链表中删除节点 nd，操作前后指针。
- **`bfs()`**：
  - 遍历当前节点 t 的邻接链表（已按目标节点编号有序）。
  - 用 `while (x < to[t])` 跳过已处理的邻接项，实现与链表遍历的同步推进。
  - 若当前链表节点 x 不在邻接表中，则标记并删除。
- **输出**：先输出胜者数量，再按编号升序输出所有胜者。

## 复杂度分析

- 时间复杂度：O(n + m)，其中 m 为胜负关系总数。每个节点最多入队一次，每个邻接边和链表节点各遍历常数次。
- 空间复杂度：O(n + m)，用于邻接表和双向链表。
