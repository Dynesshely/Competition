# P3647 [APIO2014] 连珠线 题解

## 题目链接

- [洛谷 P3647](https://www.luogu.com.cn/problem/P3647)

## 题目大意

给定一棵 $n$ 个节点的树，每条边有长度。游戏有两种操作：用红线连接新珠子和已添加的珠子，或者在两个由红线连接的珠子之间插入一个新珠子（将红线替换为两条蓝线）。最终分数为所有蓝线长度之和。求在所有能形成给定树结构的游戏过程中，最大可能得分。

## 解题思路

本题考察的是换根 DP（Tree DP with rerooting）。核心结论是：蓝线一定是由两个相邻的边组成，且这两条边不能都在同一方向上。换句话说，蓝线对应树中三个连续节点 $u-v-w$ 中 $u-w$ 被红线连接，而 $u-v$ 和 $v-w$ 是蓝线。

标准解法需要两次 DFS：一次任意根自底向上 DP，一次换根自顶向下 DP，计算每个节点作为根时的最优解。

代码中的实现仅完成了图的构建（链式前向星存图），并将所有边权相加作为答案输出。**注意**：本仓库中的 `.cpp` 文件为不完整的实现，并未包含完整的 DP 转移逻辑。

## 代码实现

- `maxn` = 200010，`maxe` 定义为 `maxn * (maxn / 400)`。
- 使用链式前向星存图：数组 `u`, `v`, `w` 存边信息，`first` 存每个点的第一条边编号，`ne` 存下一条边的编号。
- `add(uu, vv, ww)`：插入一条无向边（两条有向边），使用头插法。
- `main` 函数读取 $n-1$ 条边，调用 `add` 存图，同时将边权累加到 `ans` 中，最后输出 `ans`。

## 复杂度分析

- 时间复杂度：$O(n)$，仅完成了建图和边权和的计算。
- 空间复杂度：$O(n)$，链式前向星存图。
