# P6033 [NOIP 2004 提高组] 合并果子 加强版 题解

## 题目链接

- [洛谷 P6033](https://www.luogu.com.cn/problem/P6033)

## 题目大意

$n$ 堆果子，每次合并两堆，代价为两堆重量之和。求合并成一堆的最小总代价。$n \le 10^7$，$a_i \le 10^5$。

## 解题思路

经典哈夫曼合并问题。标准解法用优先队列，$O(n \log n)$，但 $n \le 10^7$ 会超时。

**关键约束**：$a_i \le 10^5$，值域很小。可用桶计数 + 双队列优化至 $O(n)$：

1. 桶数组 `cnt[x]` 统计每个值的出现次数。
2. 指针 `p` 从小到大扫描桶数组，模拟一个有序队列（原始堆）。
3. 队列 `q2` 存储合并产生的新堆（由于每次合并取最小两个，新堆值单调不减）。
4. 每次从桶指针和 `q2` 队首中取最小的两个值合并，结果推入 `q2`。

所有操作 $O(1)$，总复杂度 $O(n)$。

## 代码实现

- `get()` 函数比较桶指针当前值与 `q2` 队首，返回较小者。
- 桶指针用 `while` 跳过计数为零的值。
- 答案用 `long long`（$n=10^7, a_i=10^5$ 时答案可达 $10^{13}$ 级别）。

## 复杂度分析

- 时间复杂度：$O(n + \max a_i)$。
- 空间复杂度：$O(n + \max a_i)$，$q2$ 最多 $n-1$ 个元素。
