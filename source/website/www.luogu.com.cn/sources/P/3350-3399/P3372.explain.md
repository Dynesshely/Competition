# P3372 【模板】线段树 1 题解

## 题目链接

- [洛谷 P3372](https://www.luogu.com.cn/problem/P3372)

## 题目大意

给定一个数列，支持两种操作：1. 将某区间每个数加上 k；2. 求某区间所有数的和。

## 解题思路

本题是线段树的经典模板题，使用 **线段树（Segment Tree）** 配合 **懒标记（Lazy Tag）** 实现区间修改和区间查询。

**算法思想**：
- 线段树将整个区间递归划分为多个小区间，每个节点对应一个区间，存储该区间的和。
- 对于区间修改，不立即更新所有叶子节点，而是将修改操作暂存在当前节点的懒标记（tag）中，等到需要访问子区间时再下传（pushdown），从而将每次修改操作的时间复杂度降至 O(log N)。
- 区间查询时，若当前节点区间被完全覆盖，直接返回节点和；否则先下传懒标记，再递归查询子区间。

**关键操作**：
- `build(x, l, r)`：递归建树，叶子节点存储原始值，内部节点存储左右子节点之和。
- `pushup(x, v)`：将懒标记 v 应用到节点 x，更新 sum 和 tag。
- `pushdown(x)`：将当前节点的懒标记传给左右子节点，清除当前节点标记。
- `modify(x, l, r, v)`：区间加 v。若节点区间与目标完全匹配则直接 pushup 打标记；否则下传标记后继续递归。
- `query(x, l, r)`：区间求和，逻辑与 modify 类似。

## 代码实现

- `struct seg`：线段树节点结构体，包含左右子节点下标 ls、rs，区间端点 l、r，区间和 sum，懒标记 tag。
- `segement[MAXN << 2]`：线段树数组，开 4 倍空间确保安全。
- `summid(l, r)`：返回 `(l + r) >> 1` 即区间中点。
- `build()`：递归建树，`x << 1` 为左孩子，`(x << 1) | 1` 为右孩子。
- `pushup(x, v)`：累加区间和 `sum += v * (r - l + 1)`，并更新懒标记 `tag += v`。
- `pushdown(x)`：将当前节点 tag 分别 pushup 到左右孩子，然后当前节点 tag 清零。
- `modify(x, l, r, v)`：区间修改。先 pushdown，若完全匹配则 pushup 返回；否则根据位置递归到左/右子区间。
- `query(x, l, r)`：区间查询。先 pushdown，完全匹配则返回 sum；否则递归查询左右子区间并相加。
- `main()`：读入 n 和 m，建树后循环处理操作。op=1 为区间加，op=2 为区间求和。

## 复杂度分析

- 时间复杂度：O((N + M) log N)，每次 build / modify / query 操作均为 O(log N)。
- 空间复杂度：O(N)，线段树节点数约 4N。
