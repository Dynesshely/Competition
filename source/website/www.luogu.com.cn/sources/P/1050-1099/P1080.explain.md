# P1080 [NOIP 2012 提高组] 国王游戏 题解

## 题目链接

- [洛谷 P1080](https://www.luogu.com.cn/problem/P1080)

## 题目大意

国王和 $n$ 个大臣每人左右手各有一个数。大臣按某种顺序排成一列，每个大臣获得的金币数为排在前面所有人的左手数乘积除以自己右手数（向下取整）。求使获得金币最多的大臣所得金币最少的排列方案，输出该最小最大值。

## 解题思路

经典的贪心排序问题。考虑相邻两个人 $i$ 和 $j$，若交换二人能减少最大值，则应交换。推导得：按 $a_i \times b_i$ 从小到大排序即为最优排列。

证明（交换论证）：设 $i$ 排在 $j$ 前，前面的乘积为 $\Pi$。
- $i$ 在前时：$i$ 得 $\Pi/b_i$，$j$ 得 $\Pi \times a_i / b_j$。
- $j$ 在前时：$j$ 得 $\Pi/b_j$，$i$ 得 $\Pi \times a_j / b_i$。

要最小化 $\max$，则 $i$ 在前优于 $j$ 在前当且仅当 $\max(\Pi/b_i, \Pi a_i/b_j) \le \max(\Pi/b_j, \Pi a_j/b_i)$，等价于 $a_i \times b_i \le a_j \times b_j$。

由于数据范围 $n \le 1000, a,b < 10000$，乘积可能达到 $10^{4000}$ 级别，需要使用高精度（代码中用 $10000$ 进制）。

## 代码实现

- `Big` 类：$10000$ 进制高精度，支持 `mul(int)` 和 `div(int)`。
- 按 $a_i \times b_i$ 排序大臣。
- 维护前缀积 `prod`（从国王左手数开始乘）。
- 对每位大臣，计算 `cur = prod / b_i`，更新最大金币数 `ans`。
- 将 $a_i$ 乘入前缀积。

## 复杂度分析

- 时间复杂度：$O(n^2)$（高精度乘除），$n \le 1000$。
- 空间复杂度：$O(n)$，高精度数组。
