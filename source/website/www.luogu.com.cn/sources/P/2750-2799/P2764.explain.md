# P2764 最小路径覆盖问题 题解

## 题目链接

- [洛谷 P2764](https://www.luogu.com.cn/problem/P2764)

## 题目大意

给定一个有向无环图（DAG），求用最少的顶点不相交路径覆盖所有顶点，输出每条路径和最少路径数。

## 解题思路

本题是经典的 **DAG 最小路径覆盖** 问题，转化为 **二分图最大匹配** 求解，使用 **最大流（Dinic）** 实现。

**转化方法**：
1. 将原图的每个顶点 i 拆分为两个节点：左部节点 i 和右部节点 i + n。
2. 对于原图中的每条有向边 (u, v)，在左部 u 和右部 v+n 之间连一条容量为 1 的边。
3. 源点 S 向每个左部节点连容量为 1 的边，每个右部节点向汇点 T 连容量为 1 的边。
4. 求该网络的最大流，最大流的值即为最大匹配数。

**结论**：最小路径覆盖数 = 顶点数 n - 最大匹配数。

**路径输出**：通过并查集（DSU）将匹配上的边（即流量等于容量的边）连接的两个顶点合并到同一集合，每个集合即为一条路径。从每个集合的根节点出发，沿匹配边递归输出路径上的所有顶点。

## 代码实现

- **Dinic 模板**：与 P2762 相同，封装了 BFS 分层、DFS 多路增广及当前弧优化的 Dinic 算法。
- **建图**：
  - `S = n * 2 + 1`, `T = S + 1`。
  - 对每条原始边 (from, to)，添加 `dinic.addEdge(from, to + n, 1)`。
  - S 到每个左部节点 i：`dinic.addEdge(S, i, 1)`。
  - 每个右部节点 i+n 到 T：`dinic.addEdge(i + n, T, 1)`。
- **`getfa(p)`**：带路径压缩的并查集查找函数。
- **`print(p)`**：递归输出路径。从顶点 p 开始输出，然后在 p 的邻接边中查找 `real == true` 且 `flow == cap` 的边（即匹配边），递归输出其指向的顶点（需减去 n 映射回原编号）。
- **主逻辑**：
  1. 读入 n 和 m，建图并求最大流 `ret`。
  2. 初始化并查集 `fa[i] = i`。
  3. 遍历所有从左部出发的边，若为匹配边（`flow == cap`），则将 `j->to - n` 合并到 `i` 的集合中。
  4. 对于每个集合的根节点 `fa[i] == i`，调用 `print(i)` 输出整条路径。
  5. 最后输出 `n - ret`（最小路径覆盖数）。
- **快速 IO**：使用自定义的 `namespace io` 提供高性能读写，适合大数据量输入。

## 复杂度分析

- 时间复杂度：O(E sqrt(V)) 或 O(V E)（Dinic 在单位容量二分图上表现优秀），其中 V = 2n + 2，E = n + m + n。
- 空间复杂度：O(V + E)，邻接表存图。
