# P1334 洛谷 瑞瑞的木板 题解

## 题目链接
- [洛谷 P1334](https://www.luogu.com.cn/problem/P1334)

## 题目大意
瑞瑞需要 $n$ 根长度分别为 $l_i$ 的木板。他有一根长度等于所有 $l_i$ 之和的足够长的木板，需要经过 $n-1$ 次切割得到目标木板。切割长度为 $x$ 的木板需要消耗 $x$ 个单位的能量。求完成所有切割所需消耗的最小总能量。

## 解题思路
经典哈夫曼编码（Huffman coding）问题，将切割过程逆向思考为合并过程：
- 初始时每根目标木板视为一个独立的片段。
- 每次将两个片段合并，消耗的能量等于两片段长度之和。
- 最终所有片段合并为一整根木板，总消耗即为答案。
- 使用小根堆（优先队列）每次取出最小的两个片段进行合并，将合并后的新片段重新入堆，累加消耗。

## 代码实现
```cpp
#include <bits/stdc++.h>
using namespace std;

int                                                              n;
priority_queue<long long, vector<long long>, greater<long long>> pq;

void input() {
    scanf("%d", &n);
    for (int i = 0; i < n; ++i) {
        int x;
        scanf("%d", &x);
        pq.push(x);
    }
}

void process() {
    long long ans = 0;
    while (pq.size() > 1) {
        long long a = pq.top();
        pq.pop();
        long long b = pq.top();
        pq.pop();
        long long sum = a + b;
        ans += sum;
        pq.push(sum);
    }
    printf("%lld\n", ans);
}

int main() {
    input();
    process();
    return 0;
}
```

## 复杂度分析
- 时间复杂度：$O(n \log n)$，每次堆操作为 $O(\log n)$，共 $n-1$ 次合并。
- 空间复杂度：$O(n)$，用于存储优先队列。
