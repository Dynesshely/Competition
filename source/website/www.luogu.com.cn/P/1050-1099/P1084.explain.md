# P1084 [NOIP 2012 提高组] 疫情控制 题解

## 题目链接

- [洛谷 P1084](https://www.luogu.com.cn/problem/P1084)

## 题目大意

给定一棵以 $1$ 为根的树，边有边权。$m$ 支军队驻扎在某些节点（不在根）。军队可沿边移动，在任意非根节点建立检查点。求控制疫情的最少时间：从根到每个叶子的路径上至少有一个检查点。军队可同时移动。

## 解题思路

### 二分答案 + 贪心

答案具有单调性：若能 $T$ 时间内控制，则更长时间也能。对时间 $lim$ 二分。

### Check(lim) 的逻辑

**第一步：军队上移**

对每支军队，在 $lim$ 时间内尽量向根的方向移动（使用树上倍增 + 距离前缀和）。结果分为两类：
- 能到达根的军队：记录剩余时间 `rem = lim - dist_to_root` 和来自根的哪个子节点。
- 不能到达根的军队：停留在最高可达节点，标记该节点为"已驻扎"。

**第二步：检查根的子树**

DFS 从根的子节点出发，检查每个子树是否已被完全控制（所有叶子到根的路径上都有检查点）。若某个根的子树未完全控制，则需要从"能到达根的军队"中派一支过去驻扎。

**第三步：贪心匹配**

将"能到达根的军队"按剩余时间排序，将"未被控制的根的子节点"按到根的距离排序。贪心匹配：
- 若军队来自某个未被控制的子节点且剩余时间不够回来，优先用它覆盖自己的子节点。
- 否则双指针贪心：剩余时间大的军队配距离大的子节点。

### 关键细节

- 树上倍增：`f[x][j]` 表示 $x$ 的 $2^j$ 级祖先，`dist[x][j]` 表示对应路径长度。
- `dfs(x)` 检查以 $x$ 为根的子树是否已被军队驻扎保护。
- 若 `aTot < bTot`（可用军队 < 未控制子节点），直接不可行。

## 代码实现

- `bfs()`：预处理倍增数组。
- `check(lim)`：
  1. 每支军队上移，能到根则入 `h[]`，否则标记 `sta[x] = true`。
  2. 对根的子节点 DFS 检查控制状态。
  3. 排序 $h$ 和 $ned$，双指针贪心匹配。
- 二分 $[0, \sum w]$ 范围内的最小可行时间。

## 复杂度分析

- 时间复杂度：$O((n+m) \log n \log W)$，$n, m \le 5 \times 10^4$。
- 空间复杂度：$O(n \log n)$。
