# P1662 数7 题解

## 题目链接

- [洛谷 P1662](https://www.luogu.com.cn/problem/P1662)

## 题目大意

1337 个人围成一圈，从 1 号开始按顺序报数 1, 2, 3, ...。若某人报的数字是 7 的倍数或其十进制表示中包含数字 7，则报数方向反转。给定报的数字 X，求报出该数字的是第几号人。

## 解题思路

由于 X 最大可达 1e9，直接模拟 O(X) 会超时。本代码采用 **分段预处理 + 局部模拟** 的策略：

1. 将 [1, 1e9] 这个区间按每段 5e7 的长度划分为若干段。
2. 对每个分段起点（如 5e7, 1e8, ...），预先离线计算出该时刻的**当前位置（人的编号）**和**报数方向**，以硬编码（hardcode）的形式写入代码。
3. 运行时根据 X 的值找到所属分段，从该分段起点的已知状态开始，仅模拟 X 到分段起点之间的剩余步数，从而避免从头模拟整个 X。

模拟的核心逻辑：

- 每报一个数，当前人物编号沿当前方向移动一步（`+1` 或 `-1`）。
- 若编号超出 [1, 1337] 范围则进行循环修正（`0 -> 1337`, `1338 -> 1`）。
- 若当前报的数字是 7 的倍数或含有 7，则反转方向。

## 代码实现

- **`c7(x)`**：判断数字 x 的十进制中是否含有数字 7。
- **`ha(x, y, z, a)`**：核心模拟函数。
  - `x` / `y`：模拟的数字范围 [x, y]。
  - `z`：起始的人物编号。
  - `a`：起始方向（`1` 为正向，`-1` 为反向）。
  - 函数内循环从 x 到 y，依次更新人物编号和方向，最后打印结果。
- **`main` 函数的分段逻辑**：根据 n 的值选择调用 `ha` 的起始参数。例如：
  - `n < 5e7`：从 1 开始，人物 0（第一次加 1 后变为 1），方向正向。
  - `n < 1e8`：从 5e7 开始，人物 157，方向反向。
  - 以此类推，共覆盖到 1e9。
- 所有分段的起始人物编号（157, 547, 346, ...）通过线下单独运行模拟程序预先计算得到。

## 复杂度分析

- 时间复杂度：每次查询仅需模拟至多 5e7 步，即 O(段长)，相比直接 O(X) 大幅提升。
- 空间复杂度：O(1)，仅使用常数个变量。
