# P1446 [HNOI2008] Cards 题解

## 题目链接

- [洛谷 P1446](https://www.luogu.com.cn/problem/P1446)

## 题目大意

有 $n = S_r + S_b + S_g$ 张牌，需要染出 $S_r$ 张红色、$S_b$ 张蓝色、$S_g$ 张绿色。另有 $m$ 种洗牌法（均为 $1$ 到 $n$ 的排列），两种染色方案相同当且仅当可通过任意多次洗牌互相转化。求本质不同的染色方案数，对质数 $P$ 取模。数据范围：$\max\{S_r, S_b, S_g\} \le 20$，$m \le 60$，$m+1 < P < 100$。

## 解题思路

本题是 **Burnside 引理** 的经典应用。Burnside 引理指出：群作用下本质不同的着色数等于每个置换下不动点数的平均值。

本题中的群由 $m$ 种洗牌法加上恒等置换构成（共 $m+1$ 个置换），且题面保证群结构封闭。

对每个置换，计算其不动点个数（即在该置换下保持不变的染色方案数）：

1. 将置换分解为若干个循环。
2. 每个循环内的所有位置必须染同一种颜色（因为置换会循环移动它们）。
3. 令 $dp[r][b]$ 表示当前已用 $r$ 个红色、$b$ 个蓝色（绿色数量可由 $n - r - b$ 推出），处理到某个循环时的方案数。
4. 对每个长度为 $L$ 的循环，有三种选择：染红色（$r+L$）、染蓝色（$b+L$）、染绿色（绿色计数增加 $L$）。用二维背包 DP 转移。
5. 最终 $dp[S_r][S_b]$ 即为该置换的不动点个数。

对所有 $m+1$ 个置换的不动点个数求和，乘以 $(m+1)$ 模 $P$ 的逆元，即得答案。

## 代码实现

```cpp
#include <bits/stdc++.h>
using namespace std;

int sr, sb, sg, m, p, n;
int perm[65][65];
int dp[25][25];
int via[65];
int cyc[65];

void input() {
    scanf("%d%d%d%d%d", &sr, &sb, &sg, &m, &p);
    n = sr + sb + sg;
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            scanf("%d", &perm[i][j]);
    for (int i = 1; i <= n; i++)
        perm[m + 1][i] = i;
}

int modPow(int a, int b) {
    int r = 1;
    while (b) {
        if (b & 1)
            r = r * a % p;
        a = a * a % p;
        b >>= 1;
    }
    return r;
}

int calc(int *cur) {
    memset(via, 0, sizeof(via));
    int cnt = 0;
    for (int i = 1; i <= n; i++) {
        if (via[i])
            continue;
        int len = 0, x = i;
        while (!via[x]) {
            via[x] = 1;
            x      = cur[x];
            len++;
        }
        cyc[cnt++] = len;
    }
    memset(dp, 0, sizeof(dp));
    dp[0][0] = 1;
    int tot  = 0;
    for (int i = 0; i < cnt; i++) {
        int L = cyc[i];
        int old[25][25];
        memcpy(old, dp, sizeof(dp));
        memset(dp, 0, sizeof(dp));
        for (int r = 0; r <= sr; r++) {
            for (int b = 0; b <= sb; b++) {
                int g = tot - r - b;
                if (g < 0 || g > sg)
                    continue;
                int val = old[r][b];
                if (!val)
                    continue;
                if (r + L <= sr)
                    dp[r + L][b] = (dp[r + L][b] + val) % p;
                if (b + L <= sb)
                    dp[r][b + L] = (dp[r][b + L] + val) % p;
                if (g + L <= sg)
                    dp[r][b] = (dp[r][b] + val) % p;
            }
        }
        tot += L;
    }
    return dp[sr][sb];
}

void process() {
    int ans = 0;
    for (int i = 1; i <= m + 1; i++)
        ans = (ans + calc(perm[i])) % p;
    ans = ans * modPow(m + 1, p - 2) % p;
    printf("%d\n", ans);
}

int main() {
    input();
    process();
    return 0;
}
```

## 复杂度分析

- 时间复杂度：$O((m+1) \cdot n \cdot S_r \cdot S_b)$。对每个置换分解循环 $O(n)$，DP 部分 $O(n \cdot S_r \cdot S_b)$。由于 $\max\{S_r, S_b, S_g\} \le 20$，运行效率可接受。
- 空间复杂度：$O(n + S_r \cdot S_b)$。
