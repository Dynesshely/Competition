# P1018 [NOIP 2000 提高组] 乘积最大 — 题解

## 题目大意

给定一个长度为 $N$（$N \le 40$）的数字串，插入 $K$（$K \le 6$）个乘号将其分成 $K+1$ 段，求最大乘积。

## 解题思路

### 动态规划

设 `dp[i][k]` 表示前 $i$ 个字符中插入 $k$ 个乘号所能得到的最大乘积（$k < i$）。

**状态转移**：

$$dp[i][k] = \max_{k \le j < i} \big( dp[j][k-1] \times \text{num}(j, i-1) \big)$$

其中 $\text{num}(j, i-1)$ 是从下标 $j$ 到 $i-1$（0-indexed）的子串对应的数值。

**初始状态**：$dp[i][0] = \text{num}(0, i-1)$，即不插入乘号时的数值。

**答案**：$dp[N][K]$。

### 高精度乘法

由于 $N \le 40$，最大乘积可达约 $10^{40 \times 7} \approx 10^{280}$，远超 `long long` 范围，必须使用高精度。

实现要点：

- 用 `vector<int>` 存储每一位（低索引 = 低位），每个元素存 0-9
- 乘法：经典竖式乘法，`res[i+j] += a[i] * b[j]`，最后统一处理进位
- 比较：先比较位数，位数相同则从高位向低位逐位比较

### 时间复杂度

- DP 状态 $O(N \times K)$，每次转移枚举分割点 $O(N)$，高精度乘法 $O(L^2)$（$L$ 为数字位数）
- 总体 $O(N^2 K L^2)$，$N \le 40, K \le 6, L \le 280$，足够通过

### 空间复杂度

- DP 表 $O(N \times K)$，每个 `BigInt` 约 280 字节
