# P1035 [NOIP 2002 普及组] 级数求和 题解

## 题目链接

- [洛谷 P1035](https://www.luogu.com.cn/problem/P1035)

## 题目大意

已知 $S_n = 1 + \frac{1}{2} + \frac{1}{3} + \dots + \frac{1}{n}$。给定整数 $k$，求最小的 $n$ 使得 $S_n > k$。

## 解题思路

直接从 $n=1$ 开始累加调和级数，每次加上 $\frac{1}{n}$，直到和超过 $k$。由于 $k \le 15$，调和级数增长缓慢，但仍在有限步内达到目标。计算直接模拟即可。

## 代码实现

- `calculate(k)` 函数：
  - 使用 `double` 类型变量 `sn` 存储当前和。
  - 从 `index = 1` 开始循环，每次累加 `1 / index`，然后 `index++`。
  - 当 `sn > k` 时，返回 `index - 1`（因为最后一次循环中 `index` 多加了一次）。
- `main` 中读取 $K$，调用 `calculate(K)` 并输出结果。

> 注意：代码中 `sn += 1 / index` 涉及整数除法的隐式类型转换，由于 `index` 定义为 `double` 类型，所以计算结果为浮点数，保证了精度。

## 复杂度分析

- 时间复杂度：$O(n)$，其中 $n$ 为满足条件的最小整数，$n$ 随 $k$ 的增大而指数级增大（调和级数发散速度约 $\ln n$），但由于 $k \le 15$，实际运行时间可接受。
- 空间复杂度：$O(1)$，仅使用常数个变量。
