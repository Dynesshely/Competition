# P1173 [NOI2016] 网格 — 题解

## 题目大意

$n \times m$ 网格（$n,m \le 10^9$），有 $c$ 个蛐蛐（障碍）占据若干格子（$\sum c \le 10^5$），其余为跳蚤（空地）。跳蚤之间四连通。求最少将多少个跳蚤替换为蛐蛐，使存在至少两只跳蚤不连通。若无法达成输出 $-1$。

## 核心结论

**答案只有 $-1, 0, 1, 2$ 四种**。因为平面网格中总可在角落用至多 $2$ 个障碍把角落跳蚤与外界隔开。

| 答案 | 条件 |
|------|------|
| **-1** | 跳蚤总数 $< 2$，或刚好 $2$ 个且相邻 |
| **0** | 原图已不连通 |
| **1** | 连通，且存在割点（关节点） |
| **2** | 连通且无割点 |

## 解题思路

### 1. 稀疏化建图（核心难点）

$n,m \le 10^9$ 禁止建全图。关键观察：只有蛐蛐附近的格子可能成为割点或影响连通性。

对每个蛐蛐 $(x,y)$，提取其 **5×5 邻域**（Chebyshev 距离 $\le 2$）内的所有跳蚤格子作为图的节点。总计 $O(25c)$ 个节点。

**为什么是 5×5 而不是 3×3？** 3×3 会产生"假割点"——在局部看起来是割点，但因为外层有替代路径，整体并不是。向外多扩展一圈（距离 $2$）可消除这种情况。

节点标记：
- **内圈**：Chebyshev 距离 $\le 1$ 于蛐蛐的格子，标记 `isok = true`（可能成为割点）
- **外圈**：Chebyshev 距离 $=2$ 的格子，仅用于保证图的连通性正确

建边：对所有提取的跳蚤节点，四连通相邻的连边。

### 2. 连通性判定

**错误做法**：直接判断建出的图是否连通。因为可能存在建出的图不连通但原图实际连通的情况。

**正确做法**：

1. 对提取出的跳蚤格做**四连通 BFS**，标记每个格子的连通分量编号
2. 对蛐蛐做**八连通 floodfill**，将连在一起的蛐蛐作为整体
3. 对每个蛐蛐块，检查其周围（8 邻域）的所有跳蚤格是否**属于同一个分量**
4. 若存在蛐蛐块周围的跳蚤分属不同分量 → 原图已不连通 → 答案 $0$

原理：如果蛐蛐块周围的跳蚤属于不同分量，说明这个蛐蛐块构成了一道屏障，真正隔开了跳蚤图。

### 3. 割点检测

对构建好的图跑 **Tarjan 算法**求割点：

- `pre[u]`：DFS 序
- `low[u]`：能回溯到的最早祖先
- 非根节点 $u$ 是割点当存在子节点 $v$ 满足 `low[v] >= pre[u]`
- 根节点是割点当子节点数 $\ge 2$

**关键**：只有 `isok = true` 的割点才算真正的割点。距离蛐蛐 $>1$ 的格子即使局部是割点，整体也未必是（远处有替代路径）。

找到任何一个 `isok && iscut` 的节点 → 答案 $1$。

### 4. 特殊情况

- **单行/单列**（$n=1$ 或 $m=1$）：一行跳蚤中割掉一个必定断开 → 答案 $1$
- **跳蚤数 $=2$**：需检查两个跳蚤是否四相邻。因为此时 $n \times m \le \sum c + 2 \le 10^5+2$，可直接遍历网格

### 5. 否则

以上条件均不满足 → 答案 $2$。

## 时间复杂度

- 节点数 $O(25c)$，边数 $O(4 \cdot 25c) = O(100c)$
- 哈希表操作 $O(1)$ 期望
- 建图 $O(c)$，BFS $O(c)$，Floodfill $O(c)$，Tarjan $O(c)$
- 总复杂度 $O(c)$，$\sum c \le 10^5$ 足够通过

## 空间复杂度

- 哈希表、邻接表、辅助数组均为 $O(c)$
