# P3006 [USACO11JAN] Bottleneck G 题解

## 题目链接
- [洛谷 P3006](https://www.luogu.com.cn/problem/P3006)

## 题目大意
给定一棵以 1 号节点为根的有向树（N <= 100000），除根节点外每个节点 i 有一条指向父节点 P_i 的边。每个节点 i 有 C_i 头奶牛，每条边有容量 M_i，即每时间单位最多 M_i 头奶牛能从 i 走到 P_i。所有奶牛最终要向根节点 1 汇集。给定 K 个查询时间 T_i，对每个 T_i 回答到该时间为止最多能有多少头奶牛到达根节点 1。

## 解题思路
将问题转化为每个子树内部的"瓶颈"计算。对于一棵子树，其内部的奶牛总量为 cow，所有出边容量之和为 cap。在理想调度下，该子树需要 cow / cap 个时间单位将奶牛全部送出。子树的合并顺序即按照这个时间从小到大进行。

算法流程：
1. 初始化每个节点的 pass 值为其入边容量减去子节点入边容量之和（即为该子树净送出能力）。
2. 将所有 pass > 0 的叶子节点按 cow / pass 的值加入小根堆。
3. 将所有查询离线并按 T 排序。
4. 不断从堆顶取出当前瓶颈时间最小的节点，将其合并到父节点中（并查集维护）。合并后父节点继承子节点的 cow 和 pass，若 pass > 0 则重新计算时间入堆。
5. 对于每个查询时间 T，处理所有瓶颈时间 <= T 的合并后，当前根节点中 cow[1] - pass[1] * T 即为答案（注意 pass[1] 为负值，该式等价于根节点已有奶牛数加上已流入的奶牛数）。

核心思想：每个子树在未被完全清空前，其流出速率受限于自身的瓶颈容量。当多个子树合并后，速率叠加，瓶颈重新计算。

## 代码实现
使用并查集维护树的合并过程，priority_queue 按瓶颈时间顺序处理合并，离线排序处理查询。

## 复杂度分析
- 时间复杂度：O(N log N + K log K)
- 空间复杂度：O(N + K)
