# P1721 [NOI2016] 国王饮水记 题解

## 题目链接

- [洛谷 P1721](https://www.luogu.com.cn/problem/P1721)

## 题目大意

$n$ 个城市各有不同水位 $h_i$（互不相同）。国王在 $1$ 号城市，至多使用 $k$ 次"连通操作"：任选若干城市，将其水箱连通，水位变为这些城市水位的平均值。求 $1$ 号城市水位的最大值，结果要求高精度（最多 $2p$ 位小数，$p \le 3000$）。

## 解题思路

### 关键结论

1. **筛选城市**：水位低于 $h_1$ 的城市永远不会被使用（连通它们只会拉低首都水位），直接丢弃。保留 $h_1$ 和所有 $\ge h_1$ 的城市。

2. **排序**：将保留的城市按水位升序排列，$h_1$ 始终在首位（最小）。每次操作必然连通一个连续后缀（可以证明连通一个区间总是优于跳跃选择）。

3. **操作次数上界**：理论上 $k$ 可达 $10^9$，但实际有效操作次数不超过 $\min(n, 14)$。原因在于每多一次操作，水位提升呈指数衰减，约 $14$ 次后精度已无差别。

### DP 模型

设 $dp[j][i]$ 表示进行 $j$ 次操作后，涉及前 $i$ 个城市时 $1$ 号城市能达到的最高水位。转移时选择上一次操作的最后断点 $t$（$t < i$）：

$$dp[j][i] = \max_{t < i} \frac{dp[j-1][t] + (pre[i] - pre[t])}{i - t + 1}$$

即在第 $j$ 次操作中连通 $[t, i]$ 范围内的所有城市。

### 斜率优化

将转移方程改写为几何形式。考虑 $(t, pre[i])$ 点与候选点 $(t-1, pre[t] - dp[j-1][t])$ 连线斜率的最大化，可用单调队列维护下凸壳，$O(nk)$ 完成 DP。

### 高精度输出

答案精度要求 $p \le 3000$。代码中实现了 `bigNum::decimal` 类（基于 NOI 官方提供的定点高精度模板），支持 $6000$ 位精度的加减乘除和比较。最后按 $2p$ 位小数输出。

## 代码实现

- **筛选与排序**：读入 $h_i$，只保留 $\ge h_1$ 的城市，排序。
- **前缀和**：`preSum[i]` 用于快速计算区间和。
- **DP 斜率优化**：`dpF[j][i]` 存 $dp[j][i]$（`double`），`dpG[j][i]` 存转移点。
  - 维护单调队列 `que[]`，队首维护最优转移点。
  - `pts[i]` 构造为 $(i-1, preSum[i] - dpF[j-1][i])$ 用于斜率计算。
  - 队首淘汰条件：当前点与队首连线斜率 < 与下一个点的斜率。
  - 队尾维护下凸性。
- **结果重构**：找到使末尾水位最大的操作次数 $u$，通过 `dpG` 回溯构造完整过程，用高精度 `decimal` 依次计算最终值。
- **末尾处理**：使用剩余操作将城市逐一合并：`(finalAns + heights[i]) / 2`。

## 复杂度分析

- 时间复杂度：$O(n \times \min(k, 14) + n \times 位数)$，DP 部分 $O(nk)$（$k \le 14$），高精度计算 $O(n \times 位数)$。
- 空间复杂度：$O(nk)$。
