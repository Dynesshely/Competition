# P1776 宝物筛选 题解

## 题目链接

- [洛谷 P1776](https://www.luogu.com.cn/problem/P1776)

## 题目大意

给定 n 种宝物，每种宝物有价值 v[i]、重量 w[i] 和数量 m[i] 件。采集车最大载重为 W，求在不超载的前提下能带走的最大总价值。

## 解题思路

这是经典的 **多重背包（Multiple Knapsack）** 问题。每种物品有有限的数量限制，不能超过 m[i] 件。

直接对每种物品枚举件数的三重循环复杂度为 O(W * sum(m[i]))，在本题的 `sum(m[i]) <= 1e5` 且 `W <= 4e4` 时仍然可能超时。

代码采用了 **二进制拆分（Binary Splitting）** 优化，将多重背包转化为 0-1 背包：

- 对每种数量为 c 的物品，将其拆分为若干"新物品"，每件的数量分别为 1, 2, 4, 8, ... 直到剩余部分。
- 例如 c = 13，拆分为 1 + 2 + 4 + 6（13 - 1 - 2 - 4 = 6）。
- 这些新物品各自的价值和重量为原始价值/重量乘以对应的系数。
- 由于任何 1..c 范围的整数都可以用这些 2 的幂次组合表示，拆分后通过 0-1 背包即可等价实现原多重背包的效果。

拆分完成后，对所有的"新物品"执行标准的 0-1 背包（容量从大到小遍历）即可。

## 代码实现

- **`cnt`**：记录拆分后新物品的总数（从 1 开始计数）。
- **二进制拆分循环**：
  - `for (int j = 1; j <= c; j <<= 1)`：不断取 2 的幂次。
  - `v[++cnt] = j * a; w[cnt] = j * b; c -= j;`：将拆分出的新物品存入数组。
  - 若 `c > 0`（剩余部分），将剩余部分作为一件新物品存入。
- **0-1 背包 DP**：
  - 外层遍历所有新物品 i（1 到 cnt）。
  - 内层对容量 j 从 m（最大载重）向下到 `w[i]` 逆序遍历。
  - 状态转移：`f[j] = max(f[j], f[j - w[i]] + v[i])`。
- 最终输出 `f[m]`。

## 复杂度分析

- 时间复杂度：O(W * sum(log m[i]))，二进制拆分将每种物品的数量 m[i] 转化为 log m[i] 个新物品，远优于 O(W * sum(m[i]))。
- 空间复杂度：O(W + sum(log m[i]))，用于 DP 数组和拆分后的物品列表。
