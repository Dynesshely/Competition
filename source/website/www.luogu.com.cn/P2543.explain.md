# P2543 [AHOI2004] 奇怪的字符串 题解

## 题目链接

- [洛谷 P2543](https://www.luogu.com.cn/problem/P2543)

## 题目大意

给定两个仅由 0 和 1 组成的字符串，求它们的最长公共子序列（LCS）的长度。

## 解题思路

本题是经典的最长公共子序列（LCS）问题，使用 **动态规划** 解决。

**状态定义**：令 `f[i+1][j+1]` 表示字符串 `a` 的前 `i` 个字符与字符串 `b` 的前 `j` 个字符的最长公共子序列长度。

**状态转移**：
- 当 `a[i] == b[j]` 时，`f[i+1][j+1] = max(f[i][j] + 1, f[i][j+1], f[i+1][j])`
- 当 `a[i] != b[j]` 时，`f[i+1][j+1] = max(f[i][j+1], f[i+1][j])`

最终 `f[a.length()][b.length()]` 即为答案。

**注意事项**：由于字符串长度可达近 10000，二维 DP 数组的空间消耗较大（约 10^8 级别），实际实现中可能需要进行空间优化（如滚动数组）。当前代码直接开 2D VLA（变长数组，GCC 扩展），适合数据规模较小的测试点。

## 代码实现

- 使用 `cin` 读入两个字符串 `a` 和 `b`。
- 定义二维数组 `f[a.length()+1][b.length()+1]`，使用 `memset` 初始化为 0。
- 双层循环遍历两个字符串，按状态转移方程填充 DP 表。
- 输出 `f[a.length()][b.length()]` 作为结果。

## 复杂度分析

- 时间复杂度：O(n × m)，其中 n、m 为两字符串长度。
- 空间复杂度：O(n × m)，可以优化至 O(min(n, m)) 使用滚动数组。
