# P1017 [NOIP 2000 提高组] 进制转换 — 题解

## 题目大意

给定一个十进制整数 $n$ 和一个负整数 $R$（$-20 \le R \le -2$），将 $n$ 转换为以 $R$ 为基数的进制表示。

## 解题思路

### 负进制转换的原理

标准的进制转换使用**短除法**：反复除以基数取余数，直到商为 0，然后将余数逆序排列即得结果。

对于正基数 $R > 0$，余数 $r$ 始终满足 $0 \le r < R$。但当 $R < 0$ 时，直接取余可能得到负数余数。

例如：$n = -15, R = -2$，在 C++ 中：
- `n % R = (-15) % (-2) = -1`（C++11 起余数符号跟随被除数）
- `n / R = (-15) / (-2) = 7`（向零取整）

但进制表示中每位数字应满足 $0 \le r < |R|$。为此，当余数为负时，向商**借 1**：

$$n = q \times R + r$$

若 $r < 0$，则：

$$n = (q + 1) \times R + (r - R)$$

此时 $r - R \ge 0$（因为 $R < 0$，所以 $-R > 0$），余数变为合法范围。

验证：$8 \times (-2) + 1 = -16 + 1 = -15$，商 $8$ 替代了原来的 $7$，余数 $1$ 替代了原来的 $-1$。✓

### 算法步骤

1. 若 `n == 0`，直接输出 `0`
2. 反复：`r = n % R`, `n = n / R`
3. 若 `r < 0`：`r -= R`, `n += 1`
4. 将 `r` 转为对应字符（0-9 或 A-Z），拼接到结果末尾
5. 当 `n == 0` 时停止，将结果字符串逆序输出

### 时间复杂度

- $O(\log_{|R|} |n|)$，即结果位数。对于题目范围（$|n| \le 37336$，$|R| \ge 2$），至多约 16 位。

### 空间复杂度

- $O(\log_{|R|} |n|)$，仅需存储结果字符串。
