# P1053 [NOIP 2005 提高组] 篝火晚会 题解

## 题目链接

- [洛谷 P1053](https://www.luogu.com.cn/problem/P1053)

## 题目大意

$n$ 个同学按 $1 \sim n$ 围成一圈，每人给出自己最希望相邻的两个同学的编号。你可以通过以下命令调整顺序：选择 $m$ 个人，将第 $b_1$ 移到 $b_2$ 的位置，$b_2$ 移到 $b_3$ 的位置，...，$b_m$ 移到 $b_1$ 的位置，代价为 $m$。求使得所有人的相邻关系满足其意愿的最小总代价，若不可能则输出 $-1$。

## 解题思路

### 构造目标圈

首先根据每个人"最希望相邻的两个同学"来构造唯一的目标圈：
1. 从 $1$ 号同学出发，选择一个邻居作为下一个。
2. 之后每一步，当前同学的两个邻居中，一个是刚来的（上一个），另一个就是下一个要去的。
3. 重复直到回到 $1$ 号，检查是否成圈且包含所有人。
4. 若在任何步骤出现矛盾（邻居不匹配、不连通、重复出现），说明无解，输出 $-1$。

### 计算最小代价

目标圈确定后，问题转化为：当前顺序（$1,2,\ldots,n$ 的圈）要通过若干轮换（cycle shift）变成目标圈，每次操作代价等于移动人数，求最小总代价。

考虑最终目标圈与初始圈的位置对应关系。假设我们不移动某些人，只移动其他人。对于某个人 $i$，如果她在目标圈中的位置偏移量为 $d$（模 $n$ 意义下），即她需要顺时针移动 $d$ 个位置，那么我们不需要为她付出代价。最大化"不需要移动"的人数，则最小代价为 $n$ 减去这个最大值。

具体做法：
1. 分别考虑顺时针方向和逆时针方向（因为目标圈可以反向）。
2. 对于每个偏移量 $d$（$0 \sim n-1$），统计有多少人恰好处于该偏移量的正确位置上。
3. 两类方案中 $\max\_cnt$ 的最大值即为最多不需要移动的人数。
4. 答案 $= n - \max\_cnt$。

## 代码实现

- 用 `adj[i][0/1]` 存储每个人希望相邻的两人编号。
- 从 $1$ 号出发构建目标圈 `target[]`：
  - 选 `adj[1][0]` 作为下一人。
  - 每步比较当前人的两个邻居，不是来者即为去者。
  - 最后一人的邻居必须包含 $1$ 号。
- 验证圈中无重复。
- 建立 `target_pos[x]` = 编号为 $x$ 的人在目标圈中的位置。
- 顺方向：`offset = (target_pos[i+1] - i + n) % n`，统计各 offset 频次。
- 反方向：`offset = (target_pos[i+1] - (n-1-i) + n) % n`，同样统计。
- 答案为 $n - \max\_cnt$。

## 复杂度分析

- 时间复杂度：$O(n)$，构建圈 $O(n)$ + 统计偏移量 $O(n)$。
- 空间复杂度：$O(n)$，存储邻接表、目标圈、位置映射和计数数组。
