# P1526 [NOI2003] 智破连环阵 题解

## 题目链接

- [洛谷 P1526](https://www.luogu.com.cn/problem/P1526)

## 题目大意

B 国有 $M$ 个武器，依次排列，初始只有 $1$ 号武器处于攻击状态。消灭第 $i$ 号武器后，第 $i+1$ 号武器在 $1$ 秒后变为攻击状态。A 国有 $n$ 个已安放的炸弹，每个炸弹可以瞬间消灭距离不超过 $k$ 的攻击状态武器。炸弹按 $a_1, a_2, \ldots$ 顺序引爆，每个持续 $5$ 分钟。求摧毁全部 $M$ 个武器所需的最少炸弹数。

## 解题思路

### 问题转化

由于武器必须按顺序激活，每次引爆一个炸弹时可以消灭**一个正处于攻击状态的武器的连续段**——即从当前第一个存活的武器开始，依次往后若干武器中在炸弹范围内的那些。

问题转化为：将武器序列划分为尽量少的段，每段存在至少一个炸弹能覆盖该段内所有武器（即对于段内每个武器，存在一个炸弹其距离 $\le k$）。

### 预处理

1. **`canHit[i][j]`**：炸弹 $i$ 是否能打到武器 $j$，即 $(u_i - x_j)^2 + (v_i - y_j)^2 \le k^2$。

2. **`nxt[i][j]`**：若当前第一个存活武器是 $j$，使用炸弹 $i$ 后能推进到的下一个武器位置（第一个打不到的武器编号），倒序递推：

   ```
   nxt[i][j] = max(j+1, nxt[i][j+1])  if canHit[i][j]
   ```

3. **`mn[j]`**：从武器位置 $j$ 开始需要的炸弹数下界（启发式剪枝用），倒序递推：

   ```
   mn[j] = min{ mn[nxt[i][j]] + 1 }   for all i with canHit[i][j]
   ```

   最终 `mn[m] = 0`。

### 搜索 + 二分图匹配

核心是 DFS + 剪枝。`doDfs(pos, segCnt)` 表示当前处理到武器 $pos$，已使用了 $segCnt$ 个炸弹段：

- **剪枝**：若 `segCnt + mn[pos] >= ans`，当前分支不可能更优，直接剪掉。

- **枚举段终点**：尝试让当前炸弹段覆盖到 `nxtPos`（从 $m$ 到 $pos+1$）。对于每个可能的终点，将满足 `canHit[i][pos]` 且 `nxt[i][pos] >= nxtPos` 的炸弹加入当前段的二分图中。

- **二分图匹配**（`doMatch`）：左边是炸弹段编号，右边是武器编号。对当前段尝试为武器找到匹配的炸弹（匈牙利算法）。若当前段成功匹配（找到增广路），递归处理下一段。

- **回溯**：恢复 `mat` 数组和 `graph` 状态。

### 为什么是二分图匹配

同一段内的多个炸弹（如果都需要的话）需要各自匹配不同武器。匈牙利算法检验当前段的武器集合是否可以被若干炸弹覆盖——每个武器匹配一个炸弹，同一个炸弹可匹配多个武器（因为在 `5$ 分钟内同一个炸弹可以消灭多个武器）。

## 代码实现

- `sqr(x)`：返回 $x^2$，用于距离比较。
- `canHit[i][j]`：使用 `bitset` 存储，按 $O(n \cdot M)$ 暴力计算。
- `nxt[][]`：倒序遍历武器，递推计算每个炸弹在每个位置的下一可达位置。
- `mn[]`：倒序 DP 计算最小炸弹数下界。
- `doDfs(pos, segCnt)`：DFS 主函数。保存/恢复 `mat[]` 快照以实现回溯。
- `doMatch(seg)`：标准匈牙利 DFS，为炸弹段 `seg` 寻找增广路。

## 复杂度分析

- 时间复杂度：最坏是指数级（搜索空间大），但 `mn[]` 剪枝极其有效。预处理 $O(n \cdot M)$，二分图匹配单次 $O(n \cdot M)$。
- 空间复杂度：$O(n \cdot M)$，存储 `canHit`、`nxt`、`graph` 等。

> 注：本题是 NOI 2003 的提交答案题，评测采用逐测试点评分。$n, M \le 100$。
