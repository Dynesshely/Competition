# P1344 [USACO4.4] 追查坏牛奶 Pollutant Control 题解

## 题目链接

- [洛谷 P1344](https://www.luogu.com.cn/problem/P1344)

## 题目大意

给定一个有向图，源点为 $1$，汇点为 $N$，每条边有停运损失 $C_i$。求一个边集使得 $1$ 到 $N$ 不连通，要求边集总损失最小，且在损失最小的前提下停运的卡车数最少。输出最小损失和最少卡车数。

## 解题思路

本题本质是**最小割**问题。最小损失等价于求从 $1$ 到 $N$ 的最小割容量。难点在于同时要求最小割中边数最少。

**技巧：容量缩放法。** 将每条边的容量从 $c$ 转换为 $c \times K + 1$，其中 $K$ 是一个足够大的常数（代码取 $114514$）。此时：

- 最小割的总容量为 $ans = minCut \times K + edgeCount$。
- 原最小损失 = $ans / K$（整除结果）。
- 最少边数 = $ans \% K$（余数）。

原理是 $K$ 大于最大可能的边数，确保损失优先级高于边数。当多个割损失相等时，边数少的割总容量更小，自然被选出。

代码使用 **HLPP（Highest Label Preflow Push，最高标号预流推进）** 算法计算最大流 / 最小割。HLPP 是最大流算法中理论复杂度最优的之一（$O(V^2 \sqrt{E})$），比 Dinic 在稠密图上更优。

- `bfs()` 从汇点反向 BFS 初始化高度标号。
- `push()` 尝试将节点盈余推送到高度低 1 的邻接点。
- `relabel()` 当无法推送时抬升节点高度。
- 使用优先队列按高度从高到低选取活跃节点（HLPP 核心优化），并配合 `gap` 优化：若某高度层节点数为 0，则高于该层的节点均视为不可达汇点，直接抬升至 $n + 1$。

## 代码实现

- 自定义 `io` 命名空间提供高性能输入输出。
- `HLPP` 命名空间封装最大流算法：
  - `INIT(n)` 初始化 $n$ 个节点。
  - `addEdge(from, to, cap)` 添加有向边及反向边。
  - `work(S, T)` 运行 HLPP 返回最大流。
- 主函数中，每条输入的边容量乘以 $114514$ 再加 $1$。
- 计算结果 `ans = HLPP::work(1, n)`，然后输出 `ans / 114514`（最小损失）和 `ans % 114514`（最少边数）。

## 复杂度分析

- 时间复杂度：$O(V^2 \sqrt{E})$，HLPP 的理论复杂度。实际运行通常很快。
- 空间复杂度：$O(V + E)$，用于存储图、高度、盈余等数据结构。
