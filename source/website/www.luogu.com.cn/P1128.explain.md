# P1128 [HNOI2001] 求正整数 题解

## 题目链接

- [洛谷 P1128](https://www.luogu.com.cn/problem/P1128)

## 题目大意

对于给定的正整数 n (1 ≤ n ≤ 5e4)，求具有恰好 n 个正因子的最小正整数 m。

## 解题思路

根据约数个数定理：若正整数 m 的质因数分解为 m = ∏ p_i^{e_i}，则 m 的约数个数为 ∏ (e_i + 1)。因此问题转化为：寻找一组指数 e_i，使得 ∏ (e_i + 1) = n，且对应的 m = ∏ p_i^{e_i} 最小（其中 p_i 按从小到大顺序取质数，因为要 m 最小，质数必然尽量小）。

使用 DFS 搜索所有可能的指数分配方案。由于 m 可能非常大，不能直接比较数值，通过比较 log(m) = ∑ e_i * log(p_i) 来确定大小关系。搜索时对 n 的每个大于 1 的因子 d，将 e_i = d - 1 分配给当前质数。

关键剪枝：
- 指数不上升（e_i >= e_{i+1}），否则交换两个指数会使 m 变小。
- 只枚举 n 的因子作为 e_i + 1，枚举 n 的所有因子并降序排列。

找到最优指数组合后，用高精度计算最终的 m。

## 代码实现

- 预处理前 60 个质数表（用于指数分配）。
- `genDivs(x)` 生成 x 的所有大于 1 的因子，降序排列。
- `dfs(idx, rem, prev)` 当前处理第 idx 个质数，剩余因子乘积为 rem，上一指数为 prev - 1。
- 用 `bestLog` 记录当前最优解的 log 值，`bestExp` 记录对应的指数。
- 搜索完成后，用 `BigInt` 类（进制 1e9 的高精度整数）计算最终结果。
- 特判 n = 1 时答案为 1。

## 复杂度分析

- 时间复杂度：O(d(n) * k)，其中 d(n) 为 n 的因子个数，k 为搜索深度（质数个数），实际运行很快。
- 空间复杂度：O(MAXP)，即质数表的长度。
