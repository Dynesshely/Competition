# P1484 种树 题解

## 题目链接

- [洛谷 P1484](https://www.luogu.com.cn/problem/P1484)

## 题目大意

在一条直线上有 $n$ 个坑，每个坑种树有获利（可能为负），不能选择相邻的两个坑，至多种 $k$ 棵树。求最大获利。

## 解题思路

本题是经典的**反悔贪心**问题，使用**堆 + 双向链表**实现。

**为什么不能直接贪心？** 如果直接每次选当前最大获利且不邻近的位置，可能出现选了第 $i$ 个，实际选 $i-1$ 和 $i+1$ 总和更大。因此需要提供**反悔机制**。

**核心思想：**
1. 将所有坑的获利放入大根堆，同时用双向链表维护每个坑左右相邻的坑的编号。
2. 每次从堆顶取出当前最大获利 $a_i$，加入答案。
3. 标记左右坑 $a_{i-1}$ 和 $a_{i+1}$ 为已删除（后续跳过）。
4. 创建一个新节点取代位置 $i$，其权值为 $a_{i-1} + a_{i+1} - a_i$，并重新入堆。
   - 如果后续选了这个新节点，相当于反悔之前选 $i$ 的决定，转而选 $i-1$ 和 $i+1$（因为收益净变化为 $a_{i-1} + a_{i+1} - a_i$）。
5. 当堆顶权值小于 $0$ 时停止（再选只会减少获利）。
6. 重复 $k$ 次或直到堆顶为负。

这样，每次选择都保留了反悔的可能，保证了最终结果的正确性。

## 代码实现

- `num[i]`：第 $i$ 个坑的获利。
- `left[i]` / `right[i]`：双向链表，分别指向左右相邻的坑。
- `book[i]`：标记坑 $i$ 是否已被合并删除。
- 优先队列 `priority_queue<node>`：大根堆，按权值 `w` 从大到小排序。
- 每次取堆顶前，先用 `while (book[q.top().id])` 跳过已被标记删除的元素。
- 取出堆顶节点 `t`，若 `t.w < 0` 则直接跳出循环。
- 令 `c = t.id`，更新 `num[c] = num[left[c]] + num[right[c]] - num[c]`，即反悔值。
- 标记 `book[left[c]]` 和 `book[right[c]]` 为 `true`，表示左右坑已被合并删除。
- 更新链表：`left[c] = left[left[c]]`、`right[c] = right[right[c]]`，并更新新邻居的指向。
- 将更新后的节点重新入堆，提供反悔机会。

## 复杂度分析

- 时间复杂度：$O((n + k) \log n)$，每个元素最多入堆出堆一次。
- 空间复杂度：$O(n)$，用于存储权值数组、链表指针、标记数组和堆。
