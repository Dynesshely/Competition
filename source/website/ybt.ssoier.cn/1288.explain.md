# 1288 三角形最佳路径问题 题解

## 题目链接

- [一本通 1288](https://ybt.ssoier.cn/problem_show.php?pid=1288)

## 题目大意

给定一个由正整数构成的数字三角形（高度 $h \le 100$），从顶部到底部，每一步只能走到正下方或右下方，求路径上数字之和的最大值。

## 解题思路

与 1258 数字金字塔完全相同的 **动态规划** 问题。

- 状态定义：`_map[i][j]` 在代码中被原地修改为从顶部到 $(i,j)$ 的最大路径和。
- 状态转移：`_map[i][j] = max(_map[i-1][j], _map[i-1][j-1]) + _map[i][j]`。
- 读入时直接原地 DP，节省空间。
- 最后取最后一行的最大值作为答案。

## 代码实现

- `_map[100][100]`：既用于存储原始三角形，也作为 DP 表原地修改。
- 二重循环读入数据，对于非第一行的元素，执行 `_map[i][j] = max(_map[i-1][j], _map[i-1][j-1]) + _map[i][j]`。
- 遍历最后一行，取最大值 `m` 并输出。

## 复杂度分析

- 时间复杂度：$O(h^2)$，$h$ 为三角形高度。
- 空间复杂度：$O(h^2)$。
