# 1212 LETTERS 题解

## 题目链接

- [一本通 1212](https://ybt.ssoier.cn/problem_show.php?pid=1212)

## 题目大意

给定一个 $R \times S$ 的大写字母矩阵，从左上角出发，每次可以上下左右移动，但不能经过之前已经走过的字母（即同一个字母不能重复经过）。求最多能经过的不同字母个数。

## 解题思路

本题是典型的 **DFS 回溯** 问题。

- 使用二维字符数组 `_map` 存储矩阵。
- 使用布尔数组 `via[26]` 标记某个字母是否已被访问过（将字母映射为 $0..25$ 的下标）。
- DFS 搜索：从 `(0,0)` 出发，当前步数 `len=1`。
  - 标记当前位置字母已访问。
  - 更新全局最大值 `counts = max(counts, len)`。
  - 向上、下、左、右四个方向尝试，如果目标格子的字母未被访问过，则递归进入，步数 `len+1`。
  - 回溯时恢复 `via` 状态（`via[_map[row][col] - 'A'] = 0`），保证走另一条分支时可以再次经过该字母。

关键点：不能重复经过的是**字母**而非格子，因此 `via` 以字母为维度，而不是以坐标为维度。

## 代码实现

- `_map[30][30]`：存储字母矩阵。
- `via[26]`：标记 26 个大写字母是否已走过。
- `search(row, col, len)`：递归搜索函数。在当前格子标记字母已访问，更新最大长度；尝试四个方向（上、右、下、左）的邻格；回溯时取消当前字母的标记。
- 主函数读入 $R, S$ 和矩阵，从 `(0,0)` 出发调用 `search(0, 0, 1)`，最后输出 `counts`。

## 复杂度分析

- 时间复杂度：最坏情况下指数级，$O(4^{R \times S})$，但由于字母种类只有 26 种且不能重复，实际搜索深度上限为 26。
- 空间复杂度：$O(R \times S + 26)$。
