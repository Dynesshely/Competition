# 1342 【例4-1】最短路径问题 题解

## 题目链接

- [一本通 1342](https://ybt.ssoier.cn/problem_show.php?pid=1342)

## 题目大意

平面中有 $n$ 个点（$n \le 100$），部分点之间有连线，通路的距离为两点间直线距离。求从起点 $s$ 到终点 $t$ 的最短路径长度，结果保留两位小数。

## 解题思路

**Dijkstra 单源最短路径算法**。

- 首先计算所有有连线的点对之间的欧几里得距离，作为边的权值。
- 由于 $n \le 100$，使用邻接矩阵 `mapp` 存储图。
- 使用 Dijkstra 算法求 $s$ 到所有点的最短距离：
  - `dis[i]` 表示从 $s$ 到 $i$ 的最短距离。
  - `vis[i]` 标记节点是否已确定最短距离。
  - 每次选择未确定节点中 `dis` 值最小的节点 $u$，将其标记为确定，然后松弛其所有邻接点。
- 最终输出 `dis[t]`（保留两位小数）。

## 代码实现

- `re[MaxN][3]`：存储每个点的坐标，`re[i][0]` 为 $x$，`re[i][1]` 为 $y$。
- `mapp[MaxN][MaxN]`：邻接矩阵，存储点与点之间的距离（初始化为 `INF`）。
- `Dijkstra(int s)`：
  - 初始化：`vis[s] = true`，`dis[i] = mapp[s][i]`，`dis[s] = 0`。
  - 外层循环 $n-1$ 次，每次选择 $dis$ 最小且未访问的节点 `tmp_pos`。
  - 标记 `vis[tmp_pos] = true`，松弛其邻接点。
- 主函数：读入坐标，构建邻接矩阵（计算直线距离），调用 Dijkstra，输出 `dis[t]`。

## 复杂度分析

- 时间复杂度：$O(n^2)$，$n \le 100$。
- 空间复杂度：$O(n^2)$，用于邻接矩阵。
