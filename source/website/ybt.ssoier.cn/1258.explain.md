# 1258 【例9.2】数字金字塔 题解

## 题目链接

- [一本通 1258](https://ybt.ssoier.cn/problem_show.php?pid=1258)

## 题目大意

给定一个数字金字塔（行数 $R \le 1000$），从顶部开始，每一步可以走到左下方或右下方，求从顶部到底部的路径中数字和的最大值。

## 解题思路

经典 **动态规划（DP）** 问题——数字三角形。

- 状态定义：`f[i][j]` 表示从顶部走到第 $i$ 行第 $j$ 列时的最大路径和。
- 状态转移：`f[i][j] = max(f[i-1][j], f[i-1][j-1]) + tower[i][j]`。因为只能从正上方 `(i-1, j)` 或左上方 `(i-1, j-1)` 走过来。
- 边界：第一行 `f[1][1] = tower[1][1]`。
- 答案：最后一行所有 `f[n][j]` 中的最大值。

可以在读入数据的同时直接计算 `f[i][j]`，无需额外存储原始三角形（但代码中仍保留了 `tower` 数组）。

## 代码实现

- `tower[n+1][n+1]`：存储原始金字塔数据。
- `f[n+1][n+1]`：DP 表格。
- 二重循环读入 `tower[i][j]` 的同时计算 `f[i][j]`：若 $i=1$ 则初始化 `f[1][1]`；否则取 `f[i-1][j]` 和 `f[i-1][j-1]` 的较大值加上 `tower[i][j]`。
- 最后遍历最后一行的 `f[n][i]` 取最大值输出。

## 复杂度分析

- 时间复杂度：$O(n^2)$，其中 $n$ 为行数。
- 空间复杂度：$O(n^2)$，用于 DP 表。
