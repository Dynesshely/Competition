# 1893 【16NOIP提高组】玩具谜题 题解

## 题目链接

- [一本通 1893](https://ybt.ssoier.cn/problem_show.php?pid=1893)

## 题目大意

有 n 个玩具小人围成一圈，每个小人面朝圈内（0）或圈外（1），并有一个职业名称。从第一个小人开始，依次执行 m 条指令，每条指令表示向左或向右数 S 个人，最终到达的小人的职业。

## 解题思路

**模拟 + 双向循环链表。** 将 n 个小人构建成一个循环双向链表，每个节点记录朝向、职业、前驱和后继。根据当前小人的朝向和指令的方向，决定是向前驱还是后继方向移动。

核心逻辑：
- 如果小人面朝圈内（`dir == 0`），向左数（`a == 0`）是逆时针方向（链表中的 `last` 方向），向右数（`a == 1`）是顺时针方向（链表中的 `next` 方向）。
- 如果小人面朝圈外（`dir == 1`），左右方向相反：向左数走 `next`，向右数走 `last`。

代码提供了两种实现方式：
1. **公式版本（`1893.cpp`）：** 使用取模运算直接计算出最终位置，不逐个移动。
2. **迭代版本（`1893-2.cpp`）：** 使用 while 循环逐个移动 `b` 步，每次移动到前驱或后继节点。

## 代码实现

- `toy` 结构体包含 `dir`（朝向）、`last`（前驱指针）、`next`（后继指针）和 `job`（职业名称）。
- 读入 n 个玩具小人时构建双向链表：`toys[last].next = i`，`toys[i].last = last`，收尾相连形成循环。
- `pos` 表示当前所在的小人下标，初始为 0（第一个小人）。
- 对于 m 条指令，根据当前小人的朝向和指令方向确定移动方向：
  - **版本 1（公式法）：** 直接根据方向计算新位置 `pos = (pos + b) % n` 或 `pos = (pos - b + n) % n`，其中方向的映射规则与朝向相关。
  - **版本 2（迭代法）：** 用 while 循环逐个走 `b` 步，根据 `dir` 和 `a` 决定走向 `toys[pos].last` 还是 `toys[pos].next`。
- 最终输出 `toys[pos].job`。

## 复杂度分析

- 时间复杂度：O(n + m × S)（迭代版本）或 O(n + m)（公式版本），公式版本更快。
- 空间复杂度：O(n)，存储 n 个玩具小人。
