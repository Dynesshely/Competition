# 1351 【例4-12】家谱树 题解

## 题目链接

- [一本通 1351](https://ybt.ssoier.cn/problem_show.php?pid=1351)

## 题目大意

给定 $N$ 个人的家族关系，第 $i$ 行给出第 $i$ 个人的所有儿子，每行以 0 结束。要求输出一个序列，使得每个人的后辈都比那个人后列出（即拓扑排序）。

## 解题思路

**拓扑排序（Topological Sorting）** 问题。

- 将每个人的父子关系看作有向边（父节点指向子节点），构建有向图。
- 统计每个节点的入度（被指向的次数）。
- 使用队列进行拓扑排序：
  - 将所有入度为 0 的节点入队。
  - 依次取出队首节点，输出该节点，将其所有子节点的入度减 1，若子节点入度变为 0 则入队。
  - 重复直到队列为空。
- 最终输出的序列即为一个合法的拓扑序。

本题的关键是将输入"第 $i$ 行的儿子"转化为图结构。注意题目中输入的是每个人的儿子，即第 $i$ 行的数字 $x$ 表示 $i$ 是 $x$ 的父亲。

## 代码实现

注意：仓库中的 `1351.cpp` 和 `1351-1.cpp` 均为不完整的实现，存在语法问题或逻辑缺陷。下面给出标准实现的关键思路：

**1351.cpp**（语法不完整）：
- 定义了 `vector<int> people[101]` 作为邻接表。
- `up(int i)` 函数为空。

- 思路是读取每个儿子 $j$，执行 `people[j].push_back(i)` 将 $i$ 作为 $j$ 的前驱，从而构建入度关系。

**1351-1.cpp**（使用 vector 不当）：
- 定义了 `vector<int> sons` 和 `sonnum[110]`。
- 试图使用 `sons[i][sonnum[i]] = tmp` 的二维数组方式访问 vector，但 `sons` 是长度为 0 的 vector，会导致越界错误。
- 该代码也是不完整的。

两种代码都体现了拓扑排序中**构建邻接表**和**统计入度**的初步思路，但都未完成拓扑排序队列处理的核心逻辑。

标准实现应使用 Kahn 算法：
1. 构建邻接表，同时统计每个节点的入度。
2. 将入度为 0 的节点入队。
3. 循环处理队列，输出节点，更新子节点入度。

## 复杂度分析

- 时间复杂度：$O(N + M)$，其中 $M$ 为父子关系总数。
- 空间复杂度：$O(N + M)$，用于存储邻接表。
