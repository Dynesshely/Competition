# OIer 注意事项

> 考前救命集锦

---

## 一、考试策略与心态

0001. 注意策略和时间分配，搞清楚比赛策略，不要在一道题上死磕
0002. 如果准备全力想一道题，请先打完其他题的暴力，确保保底分
0003. 考试的时候千万不要炸心态，考试时要只想题目
0004. 休息很重要，好好睡觉，思维会更敏捷
0005. 想到做法只要还有时间，就不要立即写，想一想有没有更简单的做法，简化或优化做法可以减少写代码的时间、降低挂分的概率
0006. 学会写对拍，考试的时候尽量对拍，提高代码能力，减少码代码时间
0007. 对拍造数据：小数据验证正确性，极限数据验证效率

---

## 二、读题与审题

0008. 看题目要仔细，读题的时候一定要结合样例，逐字比较样例
0009. 有的样例会被分成两页，要注意检查
0010. 搞清楚有向边和无向边
0011. type 0 和 1 不要打反了
0012. 如果题目要对 2^32 或 2^64 取模，要自然想到用无符号整数溢出

---

## 三、代码编写习惯

0013. 思考五分钟，代码两小时——学会写算法流程图，想清楚再写
0014. 写的时候模块化，一个函数只做一件事
0015. 变量名注意不要和关键字冲突，变量名里有数字注意不要打反（如 sum1↔sum2）
0016. 数组下标 i 和 j 千万不要搞反了，数组的下标可能是另一个数组元素的值，不要搞混
0017. 循环里给某个变量初始化时，不要放错循环位置
0018. 括号匹配要搞清楚，较长或较复杂的计算式要多打空格提高可读性
0019. max 和 min 不要搞错了，不要复制粘贴，踏踏实实手打
0020. 同一行函数从右往左调用（栈），注意参数求值顺序的副作用

---

## 四、调试技巧

0021. 学会静态查错 ——  编译通过不代表逻辑正确，逐行脑内执行一遍
0022. 输出不要肉眼对，一定要用文件比较（diff），眼睛会出错
0023. 提交前检查是否删除了所有调试输出（cerr / printf debug）
0024. 图上问题查不出错，考虑检查建图是否出错
0025. 实在不会的问题还是问一问，带着自己的思考去请教

---

## 五、语言细节与常见陷阱

### 整数与溢出

0026. 开 long long！即使没有发现负数，也可能是 long long 的问题，分析每一个变量的精度需求
0027. 取模时注意：减法取模要 (a - b + mod) % mod，乘法可能爆 long long 需要快速乘
0028. int 和 unsigned int 混用时注意隐式类型转换

### 输入输出

0029. scanf 和 printf 数据类型要对齐（%d ↔  int, %lld ↔  long long, %lf ↔  double）
0030. 记得检查 printf 的输出格式，不要认为自己输出不会写错
0031. 尽量不写快读，写的话仔细查错；快读记得处理负数
0032. 使用 cin/cout 时记得关同步 `ios::sync_with_stdio(false)`，但之后不要混用 scanf/printf
0033. 不要用 gets，用 fgets 或 cin.getline
0034. 交互题记得 fflush(stdout)

### 浮点数

0035. 比较浮点数不要用 < 或 ==，定一个 eps（如 1e-8），浮点数的误差不超过 eps 的话比较就不会出错
0036. double 可能输出 -0，需要特判处理

### 内存与初始化

0037. 爆了空间也爆零！
0038. 多测不清空，爆零两行泪——每个测试用例前确保所有数组、变量已重置
0039. 数组开太大了 memset 会 TLE
0040. 小心 memset 时可能会越界，数组大小一定不要对应错了
0041. 用滚动数组时记得清零
0042. vector 的内存不会自动释放，shrink_to_fit 可以释放全部内存

### STL

0043. 学过的东西一定要记清楚，避免会做不会用（如 STL 各容器的特性与操作复杂度）
0044. set 等 STL 容器在用 iterator 时注意容器为空或到达边界的情况
0045. STL 容器的 size() 返回无符号类型，不要用 < 0 判断
0046. 把 lower_bound 搞清楚——返回第一个 >= x 的位置，upper_bound 返回第一个 > x 的位置
0047. 结构体排序时注意重载运算符需满足严格弱序（不能出现 a < b 且 b < a 同时为真）

### 其他

0048. Linux 与 Windows 编译环境差异大：void 函数不要写成其他返回类型；有返回值的函数记得 return
0049. void 不要写成其他有返回值的函数类型（int, bool）
0050. 有时数组下标是负数不会段错误，但是会 WA
0051. 位运算优先级低于比较运算符，记得加括号

---

## 六、数据结构

0052. 线段树等数据结构空间要开够（一般开 4 倍）
0053. 线段树在 l <= mid <= r 时，区间修改要拆分成 [l, mid] 和 [mid+1, r]
0054. 没有修改就不要上主席树——能用前缀和/普通线段树解决的别想复杂
0055. 并查集常用于维护有传递性的关系（如：等于关系、连通关系）
0056. 可以染色缩点，有时朴素想法反而比复杂数据结构更优
0057. 有多个修改的题，如果没有强制在线，考虑离线，通过某种顺序执行来降低复杂度（如：莫队）
0058. 离线询问后，除了可以分治，还可以分块

---

## 七、动态规划

0059. 知道 DP 顺序时就不要记忆化，直接递推更高效
0060. DP 时要尽可能维护多的信息，但有时也可以舍去不必要信息来降低复杂度
0061. 有的时候 DP 做不来就加一维，多出那一维也许是可以优化的，即使状态看起来过不了也可能能 A
0062. 一些想不到怎么转移的东西，可以在转移时同时转移另一个用来辅助转移的东西
0063. 一些分组 DP 转移想不到怎么优化时，可以先确认答案的上界，进而确定 DP 的限制范围
0064. 涉及到可以选择交换操作的题目，考虑操作对全局顺序造成的影响
0065. 当只需知道当前元素与前面元素的关系，而前面元素内部关系无所谓时，只需维护一个集合即可转移

---

## 八、图论

0066. 在遇到稠密图时，求最短路优先想到 Floyd
0067. SPFA 容易被卡，非负权图尽量用 Dijkstra
0068. 单元最短路构成了一棵最短路树，某些问题可以利用这个性质
0069. 当在图上可以做出一些改变图结构的决策时，或点之间的可达关系由另一个上限很小的权值决定时，可以用分层图
0070. 遇到环的问题一般都要断开，或者考虑缩点
0071. 给定序列确定顺序时，可以看成图上问题，点与点之间的前后关系可以看成有向边，用拓扑排序解决
0072. 有些涉及到删边的操作可以从边出发去考虑，倒着做有时能让删边变加边
0073. 一些从点出发不好考虑的问题可以从边出发，DP 或者松弛时通过贪心策略优化转移顺序

---

## 九、贪心与构造

0074. 排序型贪心一般用邻项微扰证明，树上贪心一般用决策包容性证明
0075. 涉及到前缀和的贪心，或和顺序有关的 01 背包（如国王游戏），通常从答案的计算式入手来得到排序方案
0076. 贪心的题目大力分类讨论，不要害怕
0077. 一些涉及到划分和分组的问题如果需要贪心，影响的东西太多，考虑倒着来——倒着来可以保证某一条限制已经合法
0078. 有些题目与操作顺序无关，找到不变量是破题关键

---

## 十、数学与计算几何

0079. 计数问题想到容斥
0080. 一些题目的答案可能有单调性，除了普通单调外，也可能是单峰函数（可三分）
0081. 倍增算贡献时要先算贡献，再将点往上跳
0082. 曼哈顿距离可以转成切比雪夫距离，反之亦然
0083. 求凸包等计算几何的题目可以想到极角排序，借此确定顺序
0084. 向量 a 叉乘向量 b 为正代表逆时针，为负代表顺时针——不要搞反了
0085. 浮点数的整次幂用快速幂，不要用 pow() ——  pow() 返回浮点数可能有精度问题
0086. 字符串哈希记得取模，双哈希更安全
0087. 一些类似整除分块要一块一块查询的时候（比如查前缀和），注意第一个块和最后一个块是否越界

---

## 十一、优化技巧与思想

0088. 要善于发现问题中隐藏的单调性，这可能大大降低复杂度
0089. 如果没有单调性，要想到去维护——对于有很多修改的题，如果没有强制在线，可以考虑离线处理
0090. 不在线询问的题都该想一想离线；一些不能离线的题目，分析限制，反过来之后限制可能交换
0091. 矩阵型计数题目可以考虑按行分治，有时也可以按列分治，两者交错进行可优化复杂度
0092. 一些维护起来比较复杂的关系可以考虑维护其逆关系
0093. 一些题目从问题入手，分析问题的限制条件，将其抽象出来，观察本质，发现本质瓶颈在哪里再解决
0094. 一些题目中的限制如果是必须为某个数 a，也可以看成要同时满足 >= a 和 <= a，二分答案时尤其有用
0095. 区间之间如果不相交、只包含，可以考虑转化成树上问题
0096. 学会打表——找规律、验证猜想、小范围暴力
0097. 书上和考试的时候多想，成绩不重要，重要的是锻炼思维
0098. 学了高级算法分析能力反而容易下降，一定要多想！多分析！不要轻易看题解
0099. 板子要复习和巩固，不要会用但写不对——会思路和能一次写对是两回事
0100. 树形 DP 注意递归深度可能导致爆栈，深度过大时考虑手写栈或换根 DP

---

## 十二、二分与搜索

0101. 二分时注意死循环：`mid = (l + r) >> 1` 配合 `l = mid + 1`；`mid = (l + r + 1) >> 1` 配合 `r = mid - 1`
0102. BFS 队列里是可以放多个起点的（多源 BFS）
0103. 博弈的题目尽量从终态往前考虑，尽可能将终态简化，在考虑策略时大力分类讨论
